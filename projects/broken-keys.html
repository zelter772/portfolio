Broken Keys

Real-time competitive word game

An idea I had for a fast-paced 1v1 word game where the keyboard itself works against you. Built as a full-stack mobile-first PWA with real-time multiplayer infrastructure, designed to feel responsive and game-like rather than like a typical web app.

Role Founder & developer
Stack React, Vite, Supabase, Zustand
Type Personal venture
Status Live
The idea

Most word games are slow and solitary. I wanted something with real stakes — a 1v1 format where keys physically break as you play, forcing you to adapt your vocabulary mid-match. The core challenge was making it feel instant: live state sync between two players, word validation against a 267,000-word dictionary, with zero perceptible lag.

I built it mobile-first because that's where people actually play word games — during commutes, in waiting rooms, between tasks. The PWA setup means it installs to the home screen and works offline-capable, so it feels like a native app without the App Store.

Key features

Real-time 1v1 multiplayer

Live state sync over Supabase real-time channels. Room creation, joining, and match state all handled in real time with no perceptible delay.

267k-word dictionary engine

Full SOWPODS dictionary validation running instantly on every submitted word — no lag even during fast-paced competitive play.

Three distinct game modes

Decay (keys break over time), Fixed (set keys disabled from the start), and Daily (one attempt, same keyboard for everyone) — each with its own strategy.

Customizable rule sets

Decay speed, round duration, minimum word length, streak multipliers, and chaos mode — all tunable per match from the settings screen.

Global leaderboard

Persistent score tracking across sessions, driving long-term engagement and competitive replay value beyond single sessions.

Installable PWA

Configured with a web manifest and service workers. Installs to home screen on mobile and works offline-capable without the App Store.
