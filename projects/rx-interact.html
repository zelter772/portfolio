Rx Interact

Drug interaction reference tool for pharmacy students

A full-stack clinical reference tool built on real data, not mocked content. Pharmacy students can look up drugs, check interactions, and save personal study notes, all backed by RxNorm, openFDA, and DDInter with 19,750 seeded drug names and 140,000+ interaction pairs.

Role Solo — data pipeline, backend, frontend, design
Stack Next.js 14, TypeScript, Supabase, Tailwind
Type Personal venture
Status Live
The problem

Pharmacy students constantly cross-reference drug information across multiple disconnected tools — dosing, contraindications, warnings, and interaction data spread across databases that feel like sterile government portals. None of them are fast, none of them are designed for students, and none of them let you save your own notes alongside the clinical data.

Rx Interact consolidates this into a single fast reference tool with a clean editorial design — backed by real clinical data sources, not a curated subset or mocked content.

Technical highlights

The interesting engineering problems on this project weren't in the features, they were in the infrastructure that makes the features actually work at real scale.

Search performance — 82ms → 2ms

Initial queries hit full sequential scans on a 19,750-row table. Fixed with a text_pattern_ops expression index on lowercased names for prefix search, and corrected fuzzy search to use pg_trgm's similarity operator properly — cutting query time from 82ms to ~2ms.

Postgres-backed API cache

Every drug page originally made 2–3 live calls to RxNorm and openFDA per request. Built a Postgres api_cache table with TTL-based expiry instead of in-memory caching, because Vercel's stateless serverless functions don't persist memory between invocations.

140k interaction pair pipeline

Seeded 140,141 drug interaction pairs from DDInter 1.0 via a three-pass name-resolution process: exact match, route-suffix-stripped matching, and confidence-gated RxNorm synonym bridging with full audit logs of every bridged match.

Honest data-gap communication

The app explicitly distinguishes three outcomes: confirmed interaction, checked with nothing found, and insufficient data to say. A data gap is never presented as a false safety signal. A deliberate product decision, not just a UI detail.

Custom FDA text pipeline

Built a text-processing pipeline that strips FDA cross-reference clutter, splits flattened label text into readable bullets, handles embedded bullet characters and en-dash sub-items, and removes near-duplicate content between label summaries and body text.

Study annotation system

Sticky-note annotations with three handwriting fonts, a toggle between sticky and plain styles, autosave on blur, and full-text search across both drug names and note content, built for exam prep and clinical rotations.

What I'd build next

Multi-drug interaction checking across 3+ drugs simultaneously, a spaced-repetition quiz mode, interaction pair saving, and backfilled interaction mechanisms where source data allows. The architecture is already built to support all of these without a rebuild.
